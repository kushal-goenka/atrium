# AGENT-SETUP.md

> **This document is written for an AI agent (Claude Code, Cursor, or similar) to follow end-to-end and produce a working Atrium deployment for an organization.** Every step has explicit inputs, commands, and a verifiable success check. No human judgment calls — if a step fails, stop and surface the failure with the exact error.

If you are a human operator, read [`docs/DEPLOY.md`](DEPLOY.md) instead. That's the narrative version.

---

## 0. Inputs the agent must gather from the operator

Collect these **before running anything**. Fail loudly if any required value is missing or looks malformed.

| Variable | Required | Format | Notes |
|---|---|---|---|
| `org.name` | yes | non-empty string | e.g. "Acme Corp" |
| `org.shortName` | no | ≤ 24 chars | defaults to `org.name` |
| `org.publicUrl` | yes | `https://host[:port]` | the URL devs will point their agents at |
| `org.orgUrl` | no | URL | org homepage (linked from header) |
| `org.supportEmail` | no | email | shown to users in the "request a plugin" flow |
| `org.proposalUrl` | no | URL | e.g. a GitHub Issues template — if set, `/suggestions` links here |
| `hosting.target` | yes | `fly` \| `railway` \| `vercel` \| `self-host` | |
| `hosting.region` | yes (if fly/railway) | platform region id | |
| `db.kind` | yes | `postgres` \| `sqlite` | `sqlite` only for trials, never prod |
| `auth.mode` | yes | `open` \| `admin-password` \| `sso` | `sso` not yet supported (planned v0.2) |
| `auth.adminPassword` | required if `auth.mode == "admin-password"` | ≥ 12 random chars | store in platform secrets, never in git |
| `auth.secret` | yes | 32 random bytes hex (≥ 64 chars) | `openssl rand -hex 32` |
| `llm.provider` | no | `anthropic` \| `openai` \| `gemini` \| `litellm-proxy` \| `ollama` \| `custom` | skip to disable AI curation |
| `llm.apiKey` | required if `llm.provider` set and not `ollama` | provider-native format | |
| `llm.baseUrl` | optional | URL | override for Azure/LiteLLM/custom |
| `llm.defaultModel` | optional | provider model id | e.g. `gemma3:4b` for Ollama |
| `airgap.mode` | no | `open` \| `allowlist` \| `strict` | default `open`. `strict` means no outbound at all. |
| `airgap.allowedHosts` | required if `airgap.mode == "allowlist"` | comma-separated hosts | |
| `federated.sources` | no | array of `{ name, kind, url, trust }` | initial sources to seed |

If the operator supplies values by natural-language description ("we're a bank, air-gapped, want SSO"), normalize to the table above and **confirm the normalized plan with the operator before proceeding**.

## 1. Clone & install

```bash
git clone https://github.com/kushal-goenka/atrium.git "$WORKDIR/atrium"
cd "$WORKDIR/atrium"
corepack enable
pnpm install --frozen-lockfile
```

**Success**: `pnpm install` exits 0, `node_modules/` exists, `pnpm --version` reports ≥ 9.15.

---

## 2. Assemble the environment file

Generate `.env` (for compose) and/or `.env.local` (for dev) from the inputs:

```bash
cat > .env.local <<EOF
# atrium generated by agent on $(date -u +%Y-%m-%dT%H:%M:%SZ)
NODE_ENV=production
DATABASE_URL="${DB_URL}"                       # postgres://... or file:./prisma/prod.db
AUTH_SECRET="${AUTH_SECRET}"                   # from auth.secret (32+ hex chars)

ATRIUM_PUBLIC_URL="${ORG_PUBLIC_URL}"
ATRIUM_ORG_NAME="${ORG_NAME}"
ATRIUM_ORG_SHORT_NAME="${ORG_SHORT_NAME:-${ORG_NAME}}"
ATRIUM_ORG_URL="${ORG_URL}"
ATRIUM_SUPPORT_EMAIL="${SUPPORT_EMAIL}"
ATRIUM_PROPOSAL_URL="${PROPOSAL_URL}"

ATRIUM_AUTH_MODE="${AUTH_MODE}"                # open | admin-password
ATRIUM_ADMIN_PASSWORD="${ADMIN_PASSWORD}"       # only if AUTH_MODE=admin-password

ATRIUM_AIRGAP="${AIRGAP_MODE:-open}"
ATRIUM_ALLOWED_HOSTS="${ALLOWED_HOSTS}"

OTEL_EXPORTER_OTLP_ENDPOINT="${OTLP_ENDPOINT}"  # optional
OTEL_SERVICE_NAME=atrium
EOF
```

Validation rules the agent **must** enforce:

- `AUTH_SECRET.length >= 64` hex chars (32 bytes) — else `openssl rand -hex 32 > .secret && AUTH_SECRET=$(cat .secret)` and **never commit `.secret`**.
- `ATRIUM_PUBLIC_URL` starts with `https://` for any non-local deployment. `http://` only allowed when hostname is `localhost`.
- `ATRIUM_ADMIN_PASSWORD.length >= 12` if set; reject shorter values.
- If `ATRIUM_AIRGAP=allowlist`, `ATRIUM_ALLOWED_HOSTS` must be non-empty.

**Success**: `node --env-file=.env.local -e "console.log(process.env.ATRIUM_ORG_NAME)"` prints the expected org name.

---

## 3. Initialize the database

```bash
pnpm exec prisma generate
pnpm exec prisma db push --skip-generate
pnpm db:seed
```

Postgres-specific: ensure the `DATABASE_URL` database exists and the user has `CREATE` privileges.

**Success**: `pnpm db:seed` prints `✓ 4 roles` and `✓ 3 sources` and exits 0.

---

## 4. Register the LLM provider (if `llm.provider` was set)

Skip this step entirely if the operator didn't want AI curation.

The agent seeds the provider directly into the DB rather than asking a human to fill in the UI. This lets the deployment come up curation-ready.

```bash
pnpm exec tsx <<'TSX'
import { prisma } from "./lib/prisma";
import { encryptSecret } from "./lib/crypto";

const apiKey = process.env.LLM_API_KEY ?? "local-no-auth";
const provider = process.env.LLM_PROVIDER!;
const displayName = process.env.LLM_DISPLAY_NAME ?? `${provider} (bootstrap)`;
const baseUrl = process.env.LLM_BASE_URL || null;
const defaultModel = process.env.LLM_DEFAULT_MODEL || null;

await prisma.providerConfig.upsert({
  where: { provider },
  create: {
    provider,
    displayName,
    apiKeyCipher: encryptSecret(apiKey),
    apiKeyTail: apiKey.slice(-4),
    baseUrl,
    defaultModel,
    enabled: true,
  },
  update: {
    displayName,
    apiKeyCipher: encryptSecret(apiKey),
    apiKeyTail: apiKey.slice(-4),
    baseUrl,
    defaultModel,
    enabled: true,
  },
});
console.log("provider seeded:", provider);
TSX
```

**For Ollama** (`llm.provider == "ollama"`): also ensure the Ollama sidecar is running and a model is pulled:

```bash
# If hosting.target == "fly" or "self-host":
docker compose --profile ollama up -d
docker compose exec ollama ollama pull "${LLM_DEFAULT_MODEL:-gemma3:4b}"
```

**Success**: calling `POST /api/admin/providers/test` (after step 7) returns `{ ok: true, sample: "pong" }`. If not, re-check `apiKey` and `baseUrl`.

## 5. Seed federated sources (if `federated.sources` provided)

For each source in the operator's list:

```bash
pnpm exec tsx <<TSX
import { prisma } from "./lib/prisma";

const sources = ${JSON.stringify(FEDERATED_SOURCES)};
for (const s of sources) {
  await prisma.source.upsert({
    where: { key: s.key },
    create: s,
    update: s,
  });
}
TSX
```

`s` schema: `{ key: string, name: string, kind: "git" | "http" | "local", url?: string, trust: "official" | "verified" | "community" | "internal" }`.

The `official-reference`, `openai-store`, `community-curated`, and `acme-internal` sources are already seeded by `pnpm db:seed`; **do not duplicate those keys**.

**Success**: `SELECT count(*) FROM Source;` returns ≥ the number of sources the operator asked for.


If build fails with "Cannot find module": re-run `pnpm install` and `pnpm exec prisma generate`.

## 7. Deploy to the chosen target

Exactly one of these sections runs. The agent picks based on `hosting.target`.

### 7a. Fly.io

```bash
fly launch --no-deploy --copy-config
fly postgres create --name "${APP_NAME}-db" --region "${REGION}" --vm-size shared-cpu-1x
fly postgres attach "${APP_NAME}-db"

# Load .env.local values into Fly secrets
grep -v '^#' .env.local | grep -v '^$' | while read -r line; do
  fly secrets set "${line}" --stage
done
fly secrets deploy

fly deploy --strategy rolling
```

**Success**: `fly status` shows `deployed` and `curl "https://${APP_NAME}.fly.dev/api/health"` returns `{"status":"ok"}`.

### 7b. Railway

```bash
railway login
railway init --name "${APP_NAME}"
railway add --database postgres
# Railway injects DATABASE_URL automatically; overlay the rest:
grep -v '^#' .env.local | grep -v '^DATABASE_URL' | grep -v '^$' | while read -r line; do
  railway variables set "${line}"
done
railway up
```

**Success**: `railway status` shows the service `running`; `/api/health` returns `ok`.

### 7c. Vercel + managed Postgres

The Vercel standalone output conflicts with Next's `output: "standalone"` in `next.config.ts`. Patch it:

```bash
sed -i.bak 's|output: "standalone",|// output: "standalone",|' next.config.ts
```

Then:

```bash
vercel link --yes --project "${APP_NAME}"
# Set the env vars via the Vercel dashboard or CLI:
grep -v '^#' .env.local | grep -v '^$' | while read -r line; do
  key="${line%%=*}"; val="${line#*=}"
  echo "${val}" | vercel env add "${key}" production
done
vercel deploy --prod
```

**Success**: deployment URL returns 200 on `/api/health`.

### 7d. Self-host via Docker Compose

```bash
cp .env.local .env     # compose reads .env
docker compose up -d
docker compose exec atrium pnpm db:push
docker compose exec atrium pnpm db:seed
```

For Ollama-backed curation: `docker compose --profile ollama up -d` and `docker compose exec ollama ollama pull ${LLM_DEFAULT_MODEL:-gemma3:4b}`.

**Success**: `curl http://localhost:3000/api/health` returns `ok` with `checks.db.status == "ok"`.

## 8. Post-deploy verification

Run this **every time**, regardless of hosting target. These are the acceptance tests for a successful setup.

```bash
BASE="${ORG_PUBLIC_URL}"
curl -fsS "${BASE}/api/health" | jq -e '.status == "ok"' >/dev/null
curl -fsS "${BASE}/mkt/marketplace.json" | jq -e '.plugins | length >= 1' >/dev/null
curl -fsS "${BASE}/" >/dev/null
curl -fsS "${BASE}/suggestions" >/dev/null
```

All four must return successfully. If any fail, stop and surface the failing URL + response body to the operator.

If `auth.mode == "admin-password"`:

```bash
curl -fsS -o /dev/null -w "%{http_code}" "${BASE}/admin"    # expect 307 redirect to /login
curl -fsS -o /dev/null -w "%{http_code}" "${BASE}/login"    # expect 200
```

## 9. Issue an initial admin API token

```bash
pnpm exec tsx <<'TSX'
import { prisma } from "./lib/prisma";
import { createApiToken } from "./lib/api-auth";

const admin = await prisma.user.upsert({
  where: { email: process.env.OPERATOR_EMAIL! },
  create: { email: process.env.OPERATOR_EMAIL!, name: process.env.OPERATOR_EMAIL! },
  update: {},
});

const { token } = await createApiToken({
  userId: admin.id,
  name: "bootstrap-admin",
  scopes: ["read:catalog", "write:sources", "write:plugins"],
});
console.log("BOOTSTRAP TOKEN (store this — shown once):", token);
TSX
```

**The token is printed once and never retrievable again.** The agent should immediately hand it to the operator's secret manager (1Password, Vault, etc.) and confirm receipt before marking this step complete.

## Failure modes the agent should recognize and surface

| Symptom | Root cause | Action |
|---|---|---|
| `AUTH_SECRET must be set to a 32+ character string` at build time | `AUTH_SECRET` empty or too short | regenerate with `openssl rand -hex 32` |
| `prisma.providerConfig.upsert` throws `P2002` | Provider already seeded | upsert is idempotent; retry with `--update` |
| `/api/health` returns `checks.db.status == "down"` | Wrong DATABASE_URL or DB not migrated | re-run step 3 |
| `/mkt/marketplace.json` is empty | Seed skipped or plugin fixture not loaded | run `pnpm db:seed` |
| Ollama test returns connection refused | sidecar not running or wrong baseUrl | restart with `docker compose --profile ollama up -d`, verify `curl http://ollama:11434/api/tags` |
| Deploy succeeds but `/suggestions` 500s | Prisma migration skipped on target | exec into the app container and run `pnpm db:push` |
| `next build` fails with "Module not found: @prisma/client" | Prisma generate skipped | `pnpm exec prisma generate && pnpm build` |

## Non-goals for this playbook

The agent should **not**:

- Configure DNS or TLS certs — that's the operator's responsibility at their registrar + load balancer.
- Seed fake user rows beyond the `MOCK_USERS` demo set — production user identity requires SSO (v0.2).
- Enable `ATRIUM_AUTH_MODE=sso` until v0.2 lands.
- Touch any existing customer data in the DB — this playbook is for **new** deployments only.
- Commit the generated `.env.local` / `.env` to source control.
